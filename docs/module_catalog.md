# Fitness-Agent 模块目录说明

论文题目：《融合知识增强与动态规划的健身智能体研究——基于 Qwen2.5 大模型的实现》

本文档用于说明各系统模块的职责、输入输出、依赖关系和对应论文创新点。

## M1 用户画像模块

职责：

构建用户长期画像和动态状态画像，为智能体决策提供个性化上下文。

输入：

- 用户基本信息
- 训练目标
- 训练水平
- 健康限制
- 近期训练记录
- 主观疲劳和恢复反馈

输出：

- `AgentContext`
- 用户长期画像
- 用户动态状态画像

依赖模块：

- M8 数据存储模块
- M6 用户反馈闭环模块

对应论文创新点：

- 面向个性化健身推荐的用户状态建模。
- 将长期目标与短期恢复状态共同纳入智能体决策。

## M2 知识增强模块（RAG）

职责：

检索健身知识库，为 Qwen2.5 问答和训练推荐提供可解释知识依据。

输入：

- 用户问题
- 用户画像
- 候选训练动作
- 健康限制
- 检索关键词或语义向量

输出：

- 知识片段
- 安全约束
- 运动处方依据
- 提示词上下文

依赖模块：

- M1 用户画像模块
- M8 数据存储模块

对应论文创新点：

- 融合外部健身知识库降低大模型幻觉风险。
- 通过知识增强提升训练建议的安全性和可解释性。

## M3 训练收益评估模块

职责：

评估训练动作或训练计划的综合收益，为动态规划提供目标函数。

输入：

- 用户目标
- 当前恢复状态
- 疲劳指数
- 候选训练动作
- 历史训练反馈

输出：

- 训练收益评分
- 疲劳代价
- 风险评分
- 候选动作排序

依赖模块：

- M1 用户画像模块
- M6 用户反馈闭环模块

对应论文创新点：

- 将训练收益、疲劳控制和健康限制统一到可计算评分体系中。
- 支撑动态规划决策过程中的状态转移评价。

## M4 动态规划决策模块

职责：

将多日训练安排建模为多阶段决策问题，搜索符合约束的最优或近似最优训练路径。

输入：

- `AgentContext`
- 训练收益评分
- 疲劳代价
- 时间约束
- 健康限制
- 候选训练动作集合

输出：

- 今日推荐训练
- 未来 7 天训练计划
- 动态规划最优路径
- 推荐理由所需的决策依据

依赖模块：

- M1 用户画像模块
- M2 知识增强模块
- M3 训练收益评估模块

对应论文创新点：

- 将动态规划引入个性化健身计划生成。
- 在训练收益最大化与疲劳风险控制之间进行可解释权衡。

## M5 Qwen2.5 智能交互模块

职责：

提供自然语言交互、计划解释、训练问答和推荐理由生成。

输入：

- 用户自然语言问题
- RAG 检索结果
- 动态规划推荐结果
- 用户画像
- 反馈记录

输出：

- AI 教练回复
- 推荐解释
- 训练调整建议
- 面向前端展示的自然语言内容

依赖模块：

- M1 用户画像模块
- M2 知识增强模块
- M4 动态规划决策模块

对应论文创新点：

- 基于 Qwen2.5 构建具备解释能力的健身智能体。
- 将模型生成能力与结构化决策结果结合，提升建议可信度。

## M6 用户反馈闭环模块

职责：

收集用户训练反馈并更新用户状态，使推荐策略形成持续优化闭环。

输入：

- 训练完成率
- AI 推荐采纳率
- 主观疲劳
- RPE
- 疼痛或不适记录
- 用户补充反馈

输出：

- 更新后的动态状态画像
- 反馈特征
- 下一轮推荐约束
- 实验评价指标

依赖模块：

- M1 用户画像模块
- M8 数据存储模块

对应论文创新点：

- 建立用户反馈驱动的训练计划自适应机制。
- 支撑智能体从一次性推荐走向持续决策。

## M7 前端展示模块

职责：

提供移动端用户界面，展示智能体驾驶舱、训练计划、AI 教练、数据分析和用户设置。

输入：

- 模拟数据
- 后续 FastAPI 接口数据
- 用户点击、输入和反馈

输出：

- 可视化训练建议
- AI 聊天交互
- 数据指标展示
- 用户设置入口

依赖模块：

- M5 Qwen2.5 智能交互模块
- M4 动态规划决策模块
- M6 用户反馈闭环模块

对应论文创新点：

- 将知识增强与动态规划结果转化为可展示、可解释、可交互的移动端体验。
- 支撑答辩阶段系统演示。

## M8 数据存储模块

职责：

持久化用户画像、训练计划、反馈记录、知识库元数据和实验数据。

输入：

- 用户注册与画像数据
- 训练计划
- AI 交互日志
- 用户反馈
- 知识库文档元数据

输出：

- 用户画像查询结果
- 训练历史
- 反馈统计
- 实验分析数据

依赖模块：

- MySQL 数据库
- 后端 ORM 或数据库访问层

对应论文创新点：

- 为个性化推荐、反馈闭环和实验评价提供统一数据基础。
- 支撑系统长期运行与论文实验复现。
